# What a Functional Value-Semantic Programming Language Is

> **Basis:** analysis of `tzervas/mycelium` @ `aad96b7` (workspace v0.463.1), 2026-07-16. Code-first review, documentation second. Part of a nine-document set; see `00-mycelium-complete-analysis.md` for the combined edition.

A functional value-semantic language is one where the fundamental unit of computation is the *value* — an immutable, self-contained piece of data whose identity is determined by what it *is*, not where it lives — and where programs are compositions of functions that transform values into new values rather than mutating state in place. Because nothing is ever modified after construction, there is no aliased mutable state: two references to "the same" value are indistinguishable from two copies of it, which structurally eliminates entire defect classes (use-after-free, data races, iterator invalidation, action-at-a-distance) rather than policing them with a borrow checker or a garbage collector's runtime discipline. Assignment and parameter passing conceptually copy (implementations optimize this away via moves, sharing, or copy-on-write, since immutability makes sharing observationally invisible), equality is structural rather than referential, and evaluation can be reasoned about equationally — substituting equals for equals — which is what makes such languages unusually amenable to formal verification, deterministic parallelism, and content-addressed identity.

## Why this framing matters for Mycelium

Mycelium takes each clause of that definition and makes it load-bearing rather than incidental. Immutability plus structurally-guaranteed acyclicity license the entire three-layer memory architecture (affine ownership, garbage-free reference counting, concurrent region reclamation — see the memory-model document). Structural identity becomes literal content addressing: a definition's identity is a BLAKE3 hash of its normalized structure, types, and contracts. And equational reasoning becomes the substrate for the certification system — the guarantee lattice, swap certificates, and discharged proof obligations all rest on the fact that a value's meaning is exhausted by its structure. The language's distinctive features are not bolted onto value semantics; they are what value semantics makes cheap.
