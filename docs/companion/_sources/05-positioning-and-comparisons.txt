# Positioning: What Sets Mycelium Apart

> **Basis:** analysis of `tzervas/mycelium` @ `aad96b7` (workspace v0.463.1), 2026-07-16. Code-first review, documentation second. Part of a nine-document set; see `00-mycelium-complete-analysis.md` for the combined edition.

**Against Rust.** The same safety *outcomes* — no use-after-free, no data races, no leaks in safe code ("in safe Mycelium a memory leak is not expressible") — reached by removing mutation instead of proving borrow discipline. The cost is giving up in-place mutation as a programming model and betting the performance gap on Perceus-style reuse plus regions plus the AOT path; the recent DN-125 value-threading design shows how mutation-*shaped* code stays expressible without a reference type ever entering the model. What Mycelium has that Rust does not: representation and trustworthiness in the type system, content-addressed identity, and audit records for reclamation.

**Against the Perceus family (Koka, Lean 4, Roc).** The same reclamation lineage — precise RC over acyclic immutable values, reuse analysis, static elision — but layered *under* affine-first ownership (so the unique path pays no RC at all) and *over* concurrent region reclamation, with a never-silent audit-record requirement none of them have, and with the elision pass architecturally outside the trusted base so optimizer bugs cannot become memory unsafety.

**Against Unison.** Content addressing, but the hash includes representation descriptors and static contracts — two definitions differing only in paradigm hash differently — and identity feeds a certification and deployment system (spores, hot-inject, an OCI-backed registry) rather than primarily a codebase manager.

**Against ML/array languages and VSA/HDC libraries.** Representation and numeric trustworthiness live *in the type system*, with a proven-where-provable, bounded-where-not conversion regime — instead of in tribal knowledge about which cast loses what. The VSA story in particular is unusually rigorous: six models under one trait with a normative per-operation guarantee matrix, capacity bounds that cite their theorems, resonator factorization honestly capped at `Empirical`, and reconstruction manifests that refuse to let indexed retrieval masquerade as compositional reconstruction.

**Against verification-oriented languages.** Verification is tunable rather than totalizing: the `fast`/`balanced`/`certified` modes and the guarantee lattice let a program be exactly as audited as its owner wants, with the never-silent mode tag ensuring the choice is always disclosed. The split regime matches proof burden to what is actually provable per conversion class instead of demanding uniform proof everywhere.

**The honest overall con.** This is a research-scale bet where several load-bearing legs — atomic cross-hypha sharing, reuse increments 2–3, full AOT language coverage, the self-hosted kernel, the two-mode typing axis, value-threading's zero-copy half — are, by the project's own scrupulous tagging, still `Declared` or in-flight. But the fact that the project *tells you that itself*, per claim, with the checked basis named, is arguably its most distinctive feature of all.
