# The Mycelium Data Model

> **Basis:** analysis of `tzervas/mycelium` @ `aad96b7` (workspace v0.463.1), 2026-07-16. Code-first review, documentation second. Part of a nine-document set; see `00-mycelium-complete-analysis.md` for the combined edition.

A runtime value is `Value = Repr + Payload + Meta`, and each leg carries deliberate weight. This document walks each mechanism, why it is the way it is, and the pros and cons of each.

## Representation in the type

`Binary{width}` (sign-free per ADR-028), `Ternary{trits}` (balanced trits in {−1, 0, +1}, arbitrary-width arithmetic per ADR-029), `Dense{dim, dtype}`, and `VSA{model, dim, sparsity}` are the four closed paradigm kinds — adding a fifth requires an RFC plus ADR — while the parameter registries (`ScalarKind`, VSA model ids, `SparsityClass`) stay open. Later value forms (`Seq{T,N}`, `Bytes`, `Float`) were appended under their own decisions (RFC-0032, ADR-040) with frozen content-address tags so no existing value's identity shifted. There is no implicit conversion, no inference of representation from usage, and no silent insertion of coercions — RFC-0012's ambient-representation feature explicitly forbids both, letting a nodule declare `default paradigm Binary` once so `{8}` means `Binary{8}`, while missing cross-paradigm conversions are statically *refused* rather than synthesized. The payload variants mirror the reprs exactly (`Bits`, `Trits`, `Scalars`, `Hypervector`, `Float`, `Seq`, `Bytes`), and deserialization routes through `Value::new`, so a wire value whose payload mismatches its declared repr is rejected, never coerced.

**Pro:** a type signature tells you the physical-semantic story completely, and cross-paradigm bugs become type errors caught before execution. **Con:** annotation weight (which the ambient mitigates but does not eliminate), and a closed paradigm set that makes adding, say, posits or unums a governance event rather than a library.

## The guarantee lattice as load-bearing metadata

Every value's `Meta` carries a `GuaranteeStrength` from the lattice `Exact ⊐ Proven ⊐ Empirical ⊐ Declared`, and composition is the lattice *meet* — weakest-wins, with the semilattice laws (commutative, associative, idempotent, identity `Exact`) verified exhaustively over all 4×4×4 tuples, a complete check rather than sampling since the space is finite. The four honesty invariants are enforced *by construction* in `Meta::new`: M-I1 (`Exact ⟺ no bound`), and M-I2/I3/I4 (a bound's *basis* — a cited theorem with checked side-conditions, an empirical fit with trial counts and method named, or a bare user assertion — determines the honest strength). An inconsistent `Meta` is unrepresentable, and `Deserialize` re-runs the invariants so a malformed wire `Meta` is rejected, never trusted. Bounds themselves are typed (`ErrorBound` with ε and norm, `ProbabilityBound`, `CrosstalkBound`, `CapacityBound`), composing through ADR-010's verified-numerics kernels (affine arithmetic for error, union bound for probability), and ADR-011 makes the basis a required companion of *every* bound, not just capacity bounds. In the surface language the tag is a type-level index (language requirement LR-6): a function demanding `Exact` input is a type error to call with `Declared`.

**Pro:** "how much should I trust this number" becomes a queryable, monotone, type-visible property that no mainstream language can express, and the propagation rule (`guarantee(result) = meet(inputs…, intrinsic)`) is the *only* sanctioned derivation, so disclosure can degrade but never spuriously upgrade. **Con:** per-value metadata that must travel everywhere (space and plumbing cost); the weakest-wins rule is pessimistic by design — one `Declared` input poisons an entire pipeline's tag even if the declared bound happens to be true; and the tags are only as honest as the bases feeding them.

## Content-addressed identity

A definition's identity is `BLAKE3(normalize(structure) ‖ types-with-repr ‖ static-contract)` — the structure α-renamed to position-independent de Bruijn indices so binder names don't matter, the encoding domain-separated and length-prefixed so distinct structures can never collide by concatenation ambiguity, and every tag registry append-only so growing the language never shifts an existing address. Names, source spans, comments, and *all dynamic `Meta`* (provenance, measured sparsity, realized bounds, certification mode) are excluded from the hash — names live in a separable `Names` map, so renaming a definition does not change its identity. The acceptance criteria are explicit: two definitions differing only in representation paradigm hash *differently*; a renamed or reformatted definition hashes *identically*; identical definitions collide. Provenance is then an acyclic DAG of these hashes (`Root` or `Derived{op, inputs}`), and the `spore` — the deployable unit (ADR-013) — carries a kernel-minted content identity that tooling `germinate`s at a target, with precedents in Nix derivations, OCI digests, and Unison's by-hash code shipping.

Floats forced a real design fight here, resolved in ADR-040: NaN payload bits are canonicalized to a single positive quiet-NaN pattern at value construction, so identity never depends on platform NaN behavior (which hardware does not determine), while `+0.0`/`−0.0` stay bit-distinct in identity but IEEE-equal under derived `==` — a documented identity-versus-equality seam. Float wire forms are strings (`"1.5"`, `"-inf"`, `"NaN"`) because JSON numbers cannot carry IEEE specials and `serde_json` would serialize a non-finite `f64` as `null` — a silent loss the design refuses.

**Pro:** perfect structural deduplication, safe hot-injection of recompiled definitions (ADR-017), reproducible identity for the spore registry (ADR-037, GHCR/OCI backend), and provenance you can actually audit. **Con:** hashing normalized ASTs isn't free; the append-only tag discipline is a forever-tax on registry evolution; and the float seams (canonical NaN, signed-zero identity) are things a user must learn.

## Explicit swaps under a split verification regime

`Swap` is the only representation-changing operation, its target and policy always lexical. The verification regime is split (RFC-0002): binary↔ternary is the one genuinely bijective class — the encode/decode is pinned in `docs/spec/swaps/binary-ternary.md` down to digit semantics, chosen widths, range, and rounding (in balanced ternary, truncation *is* round-to-nearest and ties are impossible), and the injectivity lemma is discharged both by Z3 (`unsat` on `roundtrip_8x6.smt2`) and by exhaustive decision in Rust, with the lemma's own content hash pinned in `mycelium-cert`. Everything touching Dense or VSA is bounded-or-probabilistic, and the emitted certificate says which class it is. Lossy swaps can attach a *reconstruction manifest* (RFC-0003 §6) that inspectably distinguishes **indexed retrieval** (codebook + similarity + threshold — returns a stored atom; a bounded-lossy lookup, and the type system won't let you pretend it's more) from **true compositional reconstruction** (structural recipe of roles + algebraic inverse operations — able to recover *novel* combinations never stored, VSA's defining capability over a hash table). Resonator-based decoding is capped at `Empirical` by construction (invariant FR-C2) because probabilistic factorization is all it can honestly claim.

**Pro:** the exact place correctness leaks in ML and quantization pipelines becomes a typed, certified, EXPLAIN-able event, with the honest proof burden matched to what is actually provable per swap class. **Con:** ceremony — every conversion is a visible act carrying a policy reference, which is the point but is friction; and certificate emission and checking have real cost, which motivated the next mechanism.

## Tunable certification

RFC-0034/ADR-032 gate the certificate machinery behind three modes — `fast` (default), `balanced`, `certified` — resolved through the same scoping mechanism as the ambient (`global`/`phylum`/`nodule` via `@certification`). The mode tag itself is never-silent: every `Meta` records which mode produced it, and the mode is excluded from the content hash so switching modes never perturbs identity (the exclusion is by construction — the hasher never reads `Meta`). Certificate *emission* happens in `balanced` and `certified`; *checking* only in `certified`; `Empirical`/`Proven` tags are reachable only where the machinery that justifies them runs. The §13 conformance suite verifies every clause mode-parametrically *both ways* — each invariant must fire in the tiers it applies to and be correctly absent in the tiers it does not, so a `certified`-only invariant holding spuriously in `fast` is caught. A separate named opt-out (`wrapping { … }`) covers arithmetic wraparound: never-silent failability (`Option`/`Result`/`SwapError`) is the default in every mode, and wraparound is an explicit per-use-site annotation recorded in `Meta`, never an ambient default.

**Pro:** you pay for auditability only where you want it, resolving the classic verification-tax death spiral, and the pay-for-what-you-use posture is the same shape as the memory model's. **Con:** a `fast`-mode value's guarantees are weaker in ways downstream consumers must account for — the generation≠consumption split exists precisely because this is easy to get wrong.

## Physical packing is a schedule, not a type

How trits are packed — unpacked, two-bit-per-trit, five-trit-per-byte, or bitnet.cpp's I2_S/TL1/TL2 schemes — is chosen at a lowering stage and *recorded* in `Meta.physical` as an inspectable `PhysicalLayout`, never encoded in the type (DN-01, RFC-0004 §5). Values of the same logical type interoperate whether packed or not; the selection engine that picks the schedule is the same reified, EXPLAIN-able policy mechanism (RFC-0005) used for swap targets and runtime placement.

**Pro:** layout experiments — and BitNet-style ternary inference kernels specifically — don't fracture the type system, and the chosen layout is auditable after the fact. **Con:** the logical/physical seam means performance-critical code cannot statically demand a layout; it can only observe what the scheduler chose.

## Algebraic data as a separate category

A `Construct` node evaluates to a `Datum` — constructor tag plus field values — which is deliberately *not* a fifth paradigm; the runtime value the interpreter yields is the sum `CoreValue = Value | Datum`, mirrored between the L0 interpreter and the L1 evaluator so the two agree on the data fragment. A datum carries a single meet-summary guarantee (the meet of its fields' guarantees, construction adding no approximation) and *no bound*, because a structural composite is not itself an approximation — the quantitative bounds live on the leaf representation values and are drillable via provenance. `Match` is flat and checked-exhaustive (WF7), with the Maranget decision tree that compiles nested surface patterns kept as an untrusted artifact above the kernel.

**Pro:** real ADTs with exhaustiveness checking while the closed-paradigm invariant stays intact; the honesty story extends to composites without inventing fake bounds. **Con:** the summary is lossy by design — you know *something* inside is `Empirical`, and finding which leaf means walking provenance.

## Defensive boundaries

`Repr` deserialization enforces `MAX_DIM = 2^30` on every declared dimension field, because the wire forms carry dimensions as `u32` and a crafted descriptor declaring a four-billion-element hypervector is a latent over-allocation denial-of-service on the deserialize path; the rejection names the offending field, value, and cap. The check is `Exact` (a total decidable predicate); the cap value itself is honestly tagged `Declared` (a policy choice, not a fact about any value) — a small but characteristic example of the tagging discipline applied to the language's own internals.

## Current frontier (from the document tail)

Three recent decisions extend this model. **RFC-0039** (Accepted 2026-06-30) gives the physical leg an AOT enforcement story: native Dense element-wise lowering with ADR-030 quantization descriptors, native VSA bind/bundle/permute across all six models honoring ADR-031's element-space/sparsity/complex-carrier commitments, and a dynamic-VSA JIT as an explicitly-selected `ExecMode` — everything recorded in `Meta.physical` and differential-checked against the reference interpreter. **DN-126** (Accepted 2026-07-12, `Declared`/unbuilt) ratifies type-resolution *strictness* as a genuinely third orthogonal axis — distinct from certification depth and from guarantee grade — with loose duck-typed checking on the interpreted path, strict typing gating compilation unconditionally, and mechanical strictification that writes down only *principal* inferred types, surfacing non-principal holes never-silently. **DN-120** formally closed ADR-003's disclosed residual on content-addressed identity versus efficient temporary-copy mutation (solved-by-design via the `rc == 1` reuse side-condition; see the memory-model document).
