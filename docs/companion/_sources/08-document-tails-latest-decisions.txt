# The Document Tails: Latest Decisions and What They Change

> **Basis:** analysis of `tzervas/mycelium` @ `aad96b7` (workspace v0.463.1), 2026-07-16. Code-first review, documentation second. Part of a nine-document set; see `00-mycelium-complete-analysis.md` for the combined edition.

Because the corpus is append-only with explicit supersession (see the documentation-strategy document), the highest-numbered RFCs, ADRs, and DNs carry the freshest design truth. This document inventories the tails as of the analyzed checkout.

## Late RFCs (0036–0041)

**RFC-0036 — Kernel Primitives Consolidation** (Accepted 2026-06-28) ratified the single-frozen-L0-kernel position: nine of ten nodes judged irreducibly primitive, with `FixGroup`'s derivability the one flagged open question — to be examined before freeze, a derivability question rather than an Accept blocker.

**RFC-0037 — Surface Grammar Deconfliction and Layout Independence** (**Enacted** 2026-06-27, full corpus migrated, `mycelium-l1` 615 + `mycelium-fmt` 11 tests green) is the collision-free bracket-family allocation woven through the grammar document: `[…]` type arguments, `{…}` const/width parameters, `=>` as the return arrow with `->` retired, the `0t` trit-literal prefix, the `lambda` keyword, and the short paradigm-type keywords.

**RFC-0038 — Inject-Mode Security Axis** (Accepted 2026-06-29) adds a security axis to the hot-inject path; the design is ratified but explicitly not built — every mechanism claim stays `Declared` until the Implementation DoD lands.

**RFC-0039 — Native Dense and VSA Codegen** (Accepted 2026-06-30, the four §8 open questions resolved same day) decides how `Repr::Dense` element-wise operations plus the ADR-030 quantization descriptor lower natively — with quant granularity, accumulator width, and packing schedule recorded as inspectable `Meta.physical` — how `Repr::Vsa` bind/bundle/permute lower across the BSC/HRR/FHRR/MAP-I/MAP-B/SBC models honoring ADR-031's element-space/sparsity/complex-carrier commitments, how the dynamic-VSA JIT runs (data-dependent dimension, runtime model selection, cleanup/resonator loops) as an explicitly-selected `ExecMode`, and the verification contract: every native Dense/VSA operation differential-checked against the reference interpreter.

**RFC-0040 — Vec/List Literal Desugaring** (Accepted 2026-07-03, implemented and green — 1854 tests) resolved the list-literal question: `[e1, …, en]` checked against a *cons-list-shaped* user ADT desugars to the right-nested `Cons(e1, Cons(…, Nil))` chain, with a structural recognizer for "cons-list-shaped" rather than a name match, composing unambiguously with the existing `Seq{T,N}` literal.

**RFC-0041 — Recursion-Depth Safety** (**Enacted** 2026-07-05 via the promoted W0–W7 wave, PR #1155) closed the deep-recursion `SIGABRT` class: explicit work stacks, a unified fuel budget, and manual iterative `Clone`/`PartialEq`/`Drop`/hashing on the recursive types — resolving DN-84.

## Late ADRs (040–045)

**ADR-040** (scalar float value form, Route II) and **ADR-041** (MSRV → 1.96.1) are woven through the code: the binary64-only `Float` repr, canonical-NaN identity, string wire forms for IEEE specials. **ADR-042/043** (Accepted 2026-07-07) are the self-hosting commitment pair: freeze the Rust base, rewrite everything to Mycelium under the "zero foreign first-party languages" end-state, retire each Rust component only on a dual witness (Rust differential plus native `myc` toolchain), archive never delete. **ADR-044** admitted `tokio`+`axum` to the workspace *solely* for the tero HTTP front, scoped to one crate, plain HTTP on a token-scoped `127.0.0.1` floor with TLS explicitly a reverse-proxy concern — and a subsequent trim pass extracted `mycelium-tero` verbatim into its own published repo (`tzervas/tero-rs`), removing the async runtime from the workspace entirely. **ADR-045** (Accepted 2026-07-10) is the currently-governing document: the whole-project unfreeze for a bounded gap-closure window — suspending the DN-56 kernel freeze, the L0 Core IR floor, the L1 primitive/type set, the L2/L3 surface grammar freeze, and the stdlib lexicon freeze — with the ADR-042 end-state unchanged and re-freeze bound to exhausting the DN-99 residual worklist plus a re-scored green scorecard plus a follow-up ratified ADR.

## The DN gap-closure cluster (119–140)

This cluster — the output of the ADR-045 window — is a coherent program: for every Rust construct the `mycelium-transpile` crate gaps on, a DN establishes Mycelium's *native answer* under the DN-110/111 translation taxonomy (Native Equivalent / Idiomatic Remapping / Approximation / Interop Bridge), preserving value semantics and KC-3's zero-kernel-growth bar, with verify-first grounding against exact source lines.

**DN-119** set the frame: the target is reframed from "L3 grammar accepts every Rust construct verbatim" to "**L3 expresses the native answer to every construct's underlying problem**" — the honest form of full native capability — with a verified inventory separating genuine L3 grammar gaps from work misattributed to L3 that actually belongs to the kernel, runtime, or transpiler.

**DN-120** formally closed ADR-003's disclosed residual — content-addressed identity versus efficient temporary-copy mutation — as **solved-by-design**: DN-35 §5's mechanism (reuse fires only at `rc == 1`, i.e., no live alias of the old identity; the intern/hash-cons table modeled as a weak map, evicted-or-copied on reuse) *is* the coherence answer. Detection landed `Exact`; the reuse-write stays `Declared`.

**DN-123** corrected the record/named-fields framing: the surface is already substantially supported — the transpiler carries a field-name↔index map (`StructLayout`) and desugars struct literals, projection, and struct-update to positional data.

**DN-125** decided **value-threading** as the native answer to Rust's `&mut self`/`&mut T`: take the receiver by value, return the mutated value, rebind at the call site (`x.f()` → `x = f(x)`) — zero-copy via the already-ratified DN-33 static uniqueness plus DN-35 `rc == 1` reuse plus DN-120 identity coherence, correct-with-a-copy until those land. Two adversarial narrowings are held never-silently: unprovable aliasing routes to a borrow-check precondition or a uniqueness proof, never a possibly-divergent rebind; interior-`&mut`-return shapes (`get_mut`, `iter_mut`, `IndexMut`) route to Approximation/Interop-Bridge, never a fabricated value-threaded form. The mechanism was answered-by-design; only the transpiler application was genuinely open, now scoped as M-1081.

**DN-126** ratified **two-mode typing** as a genuinely third orthogonal axis — type-resolution *strictness*, distinct from certification depth (`fast`/`certified`) and from guarantee grade (the lattice). Loose mode is the existing bidirectional checker in a non-refusing posture over the unchanged repr-dynamic evaluator; strict mode is the same checker with demotion off, gating compilation unconditionally; mechanical strictification writes down only *principal* inferred types, with non-principal/undecidable holes definitionally surfaced through the DN-04 diagnostics channel, never guessed. The runnable floor (name/arity/parse/FFI refusals) stays hard even in loose mode. Zero kernel growth, and the explicit enabler for Python transpilation (PEP-484 gradual typing maps onto loose mode directly).

**DN-127 through DN-138** work the same pattern across the stdlib-trait surface: `Display`/`format!` becomes a pure value-returning render `T → Bytes` with interpolation as `bytes_concat`, dissolving the `&mut Formatter` accumulator entirely (DN-127); `#[derive(…)]` becomes a standard-derive *lowering library* built on the already-landed DN-54 `lower` facility (DN-128); `Default` becomes an `Init` prelude trait seeded like `Fuse`/`Ord3`, alongside an `Error` prelude (DN-129); parametric instance heads `impl[T] Trait for Foo[T]` monomorphize dictionary-free as a family of concrete instances with the coherence rules extended (DN-130); bounds on inherent-impl type parameters discharge by redistribution onto the lifted methods, reusing the landed fn-bound path (DN-131); of the three pattern-surface forms only struct-variant patterns are a genuine transpiler gap, with range and `@`-binding patterns routed to desugarings rather than new productions (DN-132); qualified associated calls `Type::method(…)` emit through the deterministic `{Type}__{method}` mangle already minted on the declaration side (DN-133); expression-position enum struct-variant construction closes the `struct_layout`-population gap (DN-134); the Result/Option combinator surface is verified *already native* in `lib/std` — the real gap is closure-argument lowering (DN-135); the emit dispatch gets an interfaces-first refactor of its three residual sub-dispatch axes (DN-136); `Unit` becomes a nullary-constructor ADT, the arity-0 member of the tuple family, no new kernel node and no grammar change (DN-137); and primitive/std-type derive instances become available in freshly-transpiled files via prelude-instance seeding so the landed derive rows compose `myc check`-clean code (DN-138).

**DN-140** landed as code on the analysis date (M-1106): one total, idempotent, position-independent function `valid_ident` mapping arbitrary strings to legal Mycelium identifiers — subsuming the reserved-word escape (`_kw`), the nodule-path escape, and the illegal-character class (`_u{HEX}_`) — plus an injective length-prefix mangler, with every non-identity rewrite emitting a `// Declared: renamed …` comment.

## What the tail changes about the data and memory model story

Three of these are direct amendments to the core analysis, and they are the current design frontier.

**The mutation story is now formally closed at the design level** (DN-120 + DN-125). The memory model's Layers 1–2 gain their ergonomics layer: they are not just a reclamation strategy but the mechanism that makes mutation-*shaped* code expressible in a value-semantic language without a reference type ever entering the model. The pro is that the whole `&mut` problem dissolves into machinery already committed to; the con is that the zero-copy half rests on the two still-deferred DN-33 increments, so today it is honest-copy performance — and the design says so.

**A third orthogonal axis enters the model** (DN-126). Alongside certification depth and guarantee grade, type-resolution strictness gives a fast duck-typed dev loop and a source-language-general porting path (Python first) without a new type theory. The pro is iteration speed with the compiled path's rigor untouched; the con is that it is Accepted/`Declared` (M-1077 unbuilt), and a third axis is a third thing every consumer of a value's trust story must understand.

**The physical leg of the data model is getting real** (RFC-0039 plus the codegen tail). Native Dense/VSA lowering with quantization granularity, accumulator width, and packing schedules recorded as inspectable `Meta.physical` — plus the dynamic-VSA JIT as a named `ExecMode` — means "packing is a schedule, not a type" now has an AOT enforcement story, every native operation differential-checked against the reference interpreter per the standing discipline.

The meta-observation stands even stronger at the tail than at the head: the newest documents are the most methodologically rigorous — verify-first grounding against exact source lines, self-corrections of their own task framings recorded verbatim ("no sycophancy" is a literal house rule), ratification-under-delegation with the basis recorded, and the Accepted/Enacted seam never blurred. For anyone, human or agent, trying to determine what is true now, the resolution path is: `CHANGELOG.md` top → `CURRENT-STATE.md` → `Doc-Index.md` → the highest-numbered document touching the question — which is exactly how the strategy was designed to work.
