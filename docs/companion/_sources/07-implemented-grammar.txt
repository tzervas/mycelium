# The Implemented Grammar

> **Basis:** analysis of `tzervas/mycelium` @ `aad96b7` (workspace v0.463.1), 2026-07-16. Code-first review, documentation second. Part of a nine-document set; see `00-mycelium-complete-analysis.md` for the combined edition.

The normative oracle is `docs/spec/grammar/mycelium.ebnf` (450 lines, W3C-EBNF notation per RFC-0006's deliberate rejection of ISO 14977) paired with a 57-program accept/reject conformance corpus ‚ÄĒ every accept program must parse by the grammar, every reject program must not, and the reject set includes cases like reserved-word-in-identifier-position so the "lexes as keyword, opens no production" discipline is itself tested. The implemented lexer/parser lives in `crates/mycelium-l1` (`token.rs`, `lexer.rs`, `parse.rs`), and the `Tok::` enum matches the EBNF's vocabulary one-for-one, including the reserved-not-active words held as keywords.

## Program structure

`program ::= phylum_header? nodule_block+`. A `phylum` header groups multiple nodules in one source file as pure metadata (identity stays per-nodule per ADR-003), with `pub` exports and `use a.b.X` / `use a.b.*` glob imports under deterministic shadowing ‚ÄĒ a local declaration beats an explicit `use`, which beats a glob, and a glob-versus-glob collision on a *referenced* name is a never-silent ambiguity refusal. The `;` is a **mandatory component terminator** (DN-57/M-818) ‚ÄĒ uniformly, even after `}`-closed blocks and after the nodule header itself, which makes fully whitespace-free source legal (`nodule d; fn a() => ‚Ä¶ = ‚Ä¶;`) and gives a streaming parser the guarantee that a completed component is emittable the instant `;` arrives; the terminator adds no AST node. Top-level items are unordered for name resolution: the elaborator builds the call graph over the nodule and auto-groups each strongly-connected component of two or more functions into an inspectable `FixGroup` node (a self-looping singleton becomes `Fix`; a non-recursive function inlines) ‚ÄĒ mutual recursion is inferred, never keyworded. A nodule header may carry the atomic `@std-sys` attribute, the audited FFI-floor marker that is the only context where a `wild` block is legal.

## Items

The item forms are `use`, `default paradigm` (the RFC-0012 ambient ‚ÄĒ at most one per nodule, paradigm only, never inserting a swap, elaborating to the same L0 as the longhand twin), `type` (ADT sums of constructors with `[T]` type parameters and `{N}` const/width parameters ‚ÄĒ RFC-0037's bracket-family deconfliction, which retired `<‚Ä¶>` type arguments entirely), `trait` (a named set of `fn` signatures), `impl` in both trait-instance (`impl Trait[args] for T { ‚Ä¶ }`, under global coherence plus the phylum-wide orphan rule) and inherent (`impl T { ‚Ä¶ }`, desugaring to top-level `fn`s) forms ‚ÄĒ disambiguated after the head by the `for`-versus-`{` follower, any other follower a never-silent parse error ‚ÄĒ and `fn` with optional `pub`, optional `thaw` (per-definition de-maturation inside a matured scope; RFC-0017 moved maturation itself to scope headers and manifests, so a bare `matured fn` is a parse error with a teaching diagnostic), an optional effect annotation `!{‚Ä¶}` (kernel effect kinds `retry|alloc|io|cascade|time` plus user-declared names; absence means pure, and the coverage rule is a checker obligation), and the `@tier(compiled|interpreted)` execution hint (non-semantic by NFR-7 ‚ÄĒ never a behavioral switch).

Two composition/metaprogramming items round it out. `object` groups one constructor with trait impls, `via N : Trait` field-index delegation clauses, and inherent `fn`s ‚ÄĒ pure frontend sugar desugaring to `type` + `impl` + generated forwarding impls + `fn`, zero kernel growth, `reveal`-able. And the DN-54 pair `lower Name[params] = expr` / `derive Name for T` ‚ÄĒ user-extensible generative lowering, with surface and structural checks (rule-name uniqueness, `derive` name resolution) landed and the RHS elaboration to L0 honestly flagged as a never-silent `Residual`.

## Types

The type grammar covers the four paradigm families plus their RFC-0037 D2-b short aliases (`bin`/`tern`/`emb`/`hvec`, canonicalized to the long form by `mycfmt`; `vec` was considered and rejected for colliding with `std.collections.Vec`), `Substrate{tag}` for affine external resources, `Seq{elem, len}` and `Bytes` (RFC-0032), nullary `Float` (ADR-040, IEEE-754 binary64 only at introduction, later widths append-only), tuple/product types of arity ‚Č• 2 desugaring to synthetic single-constructor data types with no new L0 node, paradigm-less shapes like `{8}` resolved by the enclosing ambient (an ill-fitting shape is an explicit `ParadigmShapeMismatch`; no ambient in scope is an `UnresolvedAmbient`), named types with `[‚Ä¶]` type arguments, and first-class function types `A => B` ‚ÄĒ right-associative, with `->` retired and met by a never-silent teaching-reject diagnostic rather than silently aliased. The signature move: `@ Exact|Proven|Empirical|Declared` guarantee annotation on a type, binding tighter than `=>` (so `A @ Exact => B` parses as `(A @ Exact) => B`), making the honesty lattice a type-level index in the concrete syntax (LR-6).

## Expressions

The expression forms: `let ‚Ä¶ in`; `if/then/else`; `match` over one scrutinee with or-patterns (`A | B => e`, alternatives required to bind the same names at the same types) and wildcard/literal/tuple/constructor patterns; `for x in xs, acc = init => body` as elaboration-defined sugar for a structurally-recursive fold, total by construction ‚ÄĒ `while`, `loop`, `break`, `continue`, and `return` are deliberately *not* forms and *not* reserved, receiving teaching diagnostics instead; `swap(e, to: T, policy: p)` with target and policy always lexical; `with paradigm P { ‚Ä¶ }` block-scope ambient overrides that fill interior tags and elaborate away ‚ÄĒ never a conversion, with unbridged cross-paradigm edges a static `MissingConversion`; `wild { ‚Ä¶ }` as the denied-by-default unsafe escape (grammatically legal anywhere so the corpus can parse it, checker-gated to `@std-sys` nodules with the `!{ffi}` effect declared, body audited-not-verified, execution staged); `spore(e)` building a reconstruction manifest; `consume e` acquiring exclusive ownership of an affine `Substrate` (type discipline checked, runtime affinity honestly a `Residual`); the structured-concurrency fragment `colony { hypha e, ‚Ä¶ }` where an orphan `hypha` is grammatically inexpressible, reference semantics is spawn-order sequentialization (RT2), the colony yields the last hypha's value, and a `hypha` may carry a `@forage(bitmask)` D-lite placement policy (semantics-free per RT3 ‚ÄĒ placement changes performance, never the observable, with an all-zero mask an explicit `NoCandidates` refusal); `fuse(a, b)` lawful semilattice merges over a type with a declared `Fuse` instance, result metadata the meet of the inputs; `reclaim(policy) { ‚Ä¶ }` attaching a reified supervision/reclamation policy to a structured scope; and `lambda(x, y) => e` closures that curry (a multi-parameter lambda desugars to nested single-parameter closures, so partial application works) and lower via Reynolds defunctionalization ‚ÄĒ no closure heap object, no new kernel node, and a zero-parameter lambda a never-silent refusal.

## Operators and literals

The symbolic operator layer (RFC-0025) is frontend-only sugar: each operator desugars to a canonical word-function call (`a + b` ‚Üí `add(a, b)`, `-a` ‚Üí `neg(a)`), so there is no L0/L1 kernel change and the word form stays valid everywhere. Precedence follows Rust's table; every binary operator is left-associative; prefix operators bind tighter than every binary. The angle-bracket operators are fully wired because RFC-0037's move of type arguments from `<‚Ä¶>` to `[‚Ä¶]` removed the collision: `<<`/`>>` sit at shift tier, `<`/`>` at comparison tier, and the two-character `<=`/`>=` glyphs are *retired* (the worst post-type-argument ambiguity) ‚ÄĒ their word forms `lte`/`gte` have no glyph and parse as ordinary calls. Application and ascription: `app_expr ::= primary ('(' args? ')')* (':' type_ref)?`, ascription binding loosest.

Literals: `0b` binary with underscores; `0t` balanced trits, most-significant-first over `+ 0 -`; `0x` bytes with an even hex-digit count enforced; string literals with a deliberately minimal escape set (`\n \t \\ \" \0 \r`, no raw newlines, and deliberately no `\xNN` because it would let a "textual" literal inject a non-UTF-8 byte) lowering to the same `Repr::Bytes` value form as hex literals, UTF-8 encoded; float literals denoting the correctly-rounded (round-to-nearest-even) binary64 value of their decimal text, with structural int-disambiguation (`1.` stays `Int` plus the path glyph; no leading-dot form) and never-silent refusals for empty exponents and literals whose rounded value is not finite (`1e999` is a lex error ‚ÄĒ a literal is a conversion boundary, so out-of-range never silently lands on ¬Īinf); and `[e1, ‚Ä¶, en]` list literals constructing `Seq{T, N}` ‚ÄĒ or, per RFC-0040, desugaring to a right-nested `Cons(e1, Cons(‚Ä¶, Nil))` chain when checked against a *cons-list-shaped* user ADT, recognized structurally rather than by name, coexisting with `Seq` without ambiguity.

## The lexicon: active versus reserved

Active fungal-themed constructs: `nodule` (module), `phylum` (library-scale grouping), `spore` (reconstruction manifest / deployable unit), `substrate` (affine external resource), `colony`/`hypha` (structured concurrency), `fuse` (semilattice merge), `reclaim` (supervised scope), `forage` (placement policy, D-lite subset), `matured`/`thaw` (maturation and its inverse), `wild` (the unsafe escape), and `germinate` (spore deployment, in tooling). Reserved-not-active ‚ÄĒ lexed as keywords so they can never silently become identifiers, but with no production consuming them: `mesh` (decentralized gossip/pub-sub overlay), `graft` (capability contract with infrastructure), `cyst` (durable checkpoint / dormant resumable form), `xloc` (explicit value trans-location), and `backbone` (priority transport path) ‚ÄĒ the RFC-0008 R2 runtime vocabulary, pre-reserved so activating each later is an append-only event. `grow` is held as reserved-but-superseded-by-`derive`. The implemented token set also carries `wrapping` (the RFC-0034 ¬ß10.1 Axis-B opt-out block, landed in `mycelium-l1` pending ratification) and the full scalar/strength/sparsity keyword vocabulary.
