## 1. Summary

`std.vsa` is the ergonomic, value-semantic home for Mycelium's hyperdimensional / vector-symbolic algebra: the per-model operations (`bind`/`unbind`, `bundle`/`superpose`, `permute`, `similarity`, `cleanup`), the encoding helpers that compose them, and the reconstruction-manifest surface (indexed retrieval, compositional reconstruction, resonator decode). Its **honesty crux** is that **every op's guarantee tag is the one the RFC-0003 §4 matrix carries for that *(model, operation)* pair** (`mycelium-vsa::matrix`, encoded by M-242) — `permute` is `Exact` for all models, `bind` is `Exact` (algebraic) for the MAP/BSC/HRR/FHRR models and **`Proven` for sparse/SBC**, MAP/BSC `unbind` is `Exact` (self-inverse), **HRR/FHRR `unbind` is `Empirical`** (the residual weak link) and **sparse/SBC `unbind` is `Proven`**, `bundle` is `Proven`/`Empirical` per model, and **cleanup / reconstruction never claim exact recovery** — a failed or low-confidence cleanup or factorization is an explicit, tagged refusal with its similarity/confidence/margin reified, never a silent best-guess. Reconstruction holds the **probabilistic-only ceiling** (FR-C2): resonator decode is capped at `Empirical` and is never `Proven`. It is a Ring-1 capability surface (RFC-0016 §4.2): it *wraps* the landed `mycelium-vsa` models, cleanup memory, and reconstruction manifest, and **adds no trusted code** (KC-3).

## 2. Scope & module boundary

- **In scope:**
  - The per-model **operation vocabulary** over a `Hypervector` of a registered `VsaModel` (RFC-0003 §3): `bind`/`unbind` (with the model's self-inverse flag), `bundle`/`superpose` (with the model's capacity-bound derivation), `permute` (and its inverse), `similarity`, and `cleanup` (the `CleanupMemory` nearest-neighbour item memory, M-132).
  - **Encoding utilities** that compose the vocabulary: role–filler binding, sequence/positional encoding via repeated `permute`+`bundle`, n-gram/set encoding, codebook (item-memory) construction — each tagged by the *composition* of its constituents' tags (meet, weakest-wins; RFC-0001 §4.7), never upgraded.
  - The **reconstruction-manifest surface** (RFC-0003 §6 / M-260, `mycelium-core::recon`): building and inspecting a manifest that distinguishes **indexed retrieval** (codebook + similarity + threshold → a *stored atom*; bounded-lossy) from **compositional reconstruction** (recipe / role schema + algebraic inverses → *novel* combinations), and the **resonator decode** path (RFC-0009 / M-350) for factoring an unknown binding product — opt-in, probabilistic-only.
- **Out of scope (and who owns it):**
  - Dense tensors / embeddings (elementwise arithmetic, norms, dot/cosine over `Dense{dim, dtype}`) → **`std.dense`** (M-518). A hypervector is a `VSA`-`Repr` value with its model's algebra; a dense embedding is a `Dense`-`Repr` value with ε-bounded numeric ops — different `Repr` kinds (RFC-0001 §4.1), different modules.
  - Content-addressing / identity hashing — the codebook and manifest digests are `ContentHash` refs → **`std.content`** (M-523, ADR-003). `vsa` *embeds* those refs (the manifest's codebooks and recipe atoms are content-addressed, RFC-0003 §6); it does not mint or define the hash.
  - The **deployable** reconstruction artifact (the `spore`: a content-addressed DAG of code + values + the manifest, shipped by hash) → **`std.spore`** (M-522, ADR-013; RFC-0003 §6 r2 scope note). `vsa` owns the reconstruction *operations* and the *manifest record*; `spore` owns *packaging the deployable*.
  - The representation change `Dense ↔ VSA` (encode/decode across `Repr` paradigms, certificate-carrying) → **`std.swap`** (M-516; RFC-0002 §5; M-231). `vsa` never re-encodes a value into another `Repr` silently.
  - The ε/δ **bound kernels + shared certificate** the capacity/crosstalk bounds route through → **`std.numerics`** (M-512; ADR-010). `vsa` *carries* the `CapacityBound`/`CrosstalkBound` the model derives; it does not define the bound algebra.
- **Ring & layering:** Ring 1, Tier A (RFC-0016 §4.2/§4.3). `std.vsa` **wraps** the landed `mycelium-vsa` surface — the `VsaModel` trait + registered models (M-130/240/241/242), the `CleanupMemory` (M-132), the encoded §4 `matrix`, and the `mycelium-core::recon`/`mycelium-vsa::recon` manifest (M-260) — and **consumes** the `ContentHash` refs (`std.content`) the manifest embeds. It defines **no new** `Repr` kind (the four kinds are closed in the kernel, RFC-0001 §4.1), no new model (models register through the `ModelId` hook, RFC-0003 §2), and **enlarges no trusted base** (KC-3). It is a certificate / EXPLAIN consumer, not a producer of trusted bounds.

## 3. Exported-op surface (design sketch)

A design sketch — enough to fix the surface and feed the guarantee matrix, not a committed grammar. Value-semantic and immutable-by-default. Ops are parameterised by a `Model: VsaModel`; a fallible op returns `Result`/`Option`; the result carries the model-and-op's guarantee tag in `Meta.guarantee` + the relevant `Meta.bound` (`CapacityBound`/`CrosstalkBound`, RFC-0001 §4.3). The op tag is **read from the §4 matrix**, never chosen here.

```
// illustrative signatures (not a committed surface)

type Hv<M: VsaModel>                  // = Value<VSA{model: M, dim: d}> (RFC-0001 §4.1); the Hypervector slot

// --- the per-model operation vocabulary (RFC-0003 §3) ---
fn bind<M>(a: Hv<M>, b: Hv<M>)            -> Hv<M>     // Exact (algebraic) for all models — RFC-0003 §4 / §4.1
fn unbind<M>(c: Hv<M>, b: Hv<M>)         -> Hv<M>     // Exact for self-inverse (MAP/BSC); Empirical for HRR/FHRR (§4.1)
fn bundle<M>(xs: &[Hv<M>])               -> Result<Hv<M>, VsaError>
    // superposition; tag per §4 (Proven MAP-I / sparse / on-expectation BSC; Empirical HRR/FHRR);
    // MAP-B nested-bundle input -> Err(NestedBundleUnsupported) (RR-13; never a silent accuracy loss)
fn permute<M>(a: Hv<M>, p: Perm)         -> Hv<M>     // Exact for ALL models (§4.1 erratum — a bijection)
fn unpermute<M>(a: Hv<M>, p: Perm)       -> Hv<M>     // Exact (inverse permutation)
fn similarity<M>(a: Hv<M>, b: Hv<M>)     -> Score     // the model's similarity primitive

// --- cleanup: indexed retrieval against an item memory (M-132) ---
fn cleanup<M>(codebook: &ItemMemory<M>, q: Hv<M>) -> Result<Match, VsaError>
    // Match { label, confidence, margin }; bounded-lossy (Clarkson Thm 16) -> Empirical.
    // confidence below threshold -> Err(BelowCleanupThreshold); ambiguous (small margin) -> Err(Ambiguous).
    // NEVER a silent low-confidence "best guess" passed off as exact (FR-C2 / RFC-0009 §5.4)

// --- encoding utilities (compositions; tag = meet of constituents, never upgraded) ---
fn bind_role<M>(role: Hv<M>, filler: Hv<M>) -> Hv<M>            // = bind
fn encode_seq<M>(items: &[Hv<M>])           -> Result<Hv<M>, VsaError>  // permute^i then bundle
fn encode_set<M>(items: &[Hv<M>])           -> Result<Hv<M>, VsaError>  // bundle of atoms

// --- reconstruction manifest (RFC-0003 §6 / M-260) ---
fn manifest_indexed<M>(codebooks: &[ContentHash], threshold: Score) -> Result<ReconInfo, VsaError>
fn manifest_compositional<M>(codebooks: &[ContentHash], recipe: Recipe) -> Result<ReconInfo, VsaError>
fn reconstruct<M>(m: &ReconInfo, s: Hv<M>) -> Result<Reconstructed, VsaError>
    // indexed -> a stored atom (bounded-lossy); compositional -> recipe + algebraic inverse

// --- resonator decode: factor an UNKNOWN binding product (RFC-0009 / M-350), opt-in, probabilistic-only ---
fn reconstruct_factors<M>(m: &ReconInfo, s: Hv<M>) -> Result<Factors, ResonatorError>
    // Empirical ceiling, NEVER Proven (FR-C2); only a clean `Converged` verdict clearing tau_lock +
    // confidence + margin yields factors; BudgetExhausted | Oscillating | Stalled | OutsideEmpiricalProfile
    // are explicit error verdicts carrying the run trace for EXPLAIN — never a returned wrong fixed point.

enum VsaError { NestedBundleUnsupported, BelowCleanupThreshold, Ambiguous,
                ModelMismatch, DimMismatch, EmptyBundle, MalformedReconstruction }
enum ResonatorError { BudgetExhausted, Oscillating, Stalled, OutsideEmpiricalProfile,
                      BelowCleanupThreshold, Ambiguous }
```

Two operands of mismatched `model` or `dim` are a mismatch error (`ModelMismatch`/`DimMismatch`) — never a silent coercion across models or dimensions. A bundle of zero items is `Err(EmptyBundle)`, never a fabricated zero vector. No op silently approximates: an approximate result *carries its tag and bound*, and a refused one *carries its diagnostic* (RFC-0013).

## 4. Guarantee matrix (the load-bearing deliverable — RFC-0016 §4.5)

Rows = exported ops, **split by model where the §4 matrix splits the tag** (the per-`(model, op)` cell is the unit of honesty). This table **mirrors the encoded `mycelium-vsa::matrix` (M-242)**, which is the authoritative reading of RFC-0003 §4 as corrected by the r3 §4.1 erratum — it is not re-derived here. "Tag" is the RFC-0001 lattice strength (`Exact ⊐ Proven ⊐ Empirical ⊐ Declared`). To be asserted in tests against `matrix.rs` once the library code lands — never prose only.

| `reconstruct_factors` (resonator) | sparse / HRR / FHRR | `Declared` (lossy / non-exact self-inverse — **not `Empirical`**, RFC-0009 §9 Q6) **[FLAG Q4]** | as above | none | yes (run trace) |

**Tag justification (VR-5 — downgrade rather than overclaim; every tag cites the RFC-0003 §4 matrix):**

- **`Exact` rows.** `bind` is a deterministic algebraic operation — `Exact` for the MAP/BSC/HRR/FHRR models (elementwise multiply / XOR / circular convolution / complex multiply), carrying **no** `Bound` (RFC-0001 M-I1); **sparse/SBC `bind` is `Proven`, not `Exact`** (its guarantee is the checked Bloom analysis, see the `Proven` rows below — `matrix.rs` `("SBC", Bind, Proven)`). `unbind` is `Exact` *only* for the self-inverse models (MAP-I/MAP-B/BSC: `g⁻¹ = g`); **sparse/SBC `unbind` is `Proven`** (likewise the Bloom-regime inverse, below). `permute`/`unpermute` are `Exact` for **all** models: a permutation is a fixed coordinate bijection that relabels components without altering the alphabet and is exactly invertible — the §4.1 erratum corrects the r2 table cell, whose "Proven; quadratic in sequence length" conflated the *operation* with **sequence-decoding error growth**, which belongs to the `bundle`/`unbind` composition, not to `permute`. (Basis: RFC-0003 §4.1, research T1.2.)
- **`Proven` rows.** `bundle` is `Proven` only where the §4 matrix cites a **non-asymptotic concentration bound** whose side-conditions are *checked* (the M-131 checked-instantiation pattern: the formula's soundness is an axiomatized cited theorem, the checker discharges the concrete `{d, k, s, m, δ}` arithmetic — RFC-0003 §5): MAP-I (Clarkson Thm 6 / Thomas), MAP-B *membership-only* (Clarkson Thm 16, with deep nesting **forbidden** under `Proven` → the RR-13 `NestedBundleUnsupported` refusal, M-240/M-242), BSC *on expectation* (Heim / Yi & Achour — explicitly the weaker on-expectation form, tagged as such), and sparse/SBC (Bloom analysis, Clarkson Thms 22–23). **Sparse/SBC `bind` and `unbind` are likewise `Proven`** (the counting-Bloom inverse is exact within the Bloom regime — `matrix.rs` `("SBC", Bind, Proven)` / `("SBC", Unbind, Proven)`), tagged `Proven` rather than `Exact` because the guarantee is the checked Bloom analysis, not a bare algebraic identity. The exact theorem constants and side-conditions are **owned by RFC-0003 §4/§5 and the encoded `matrix`**, cited not restated here.
- **`Empirical` rows.** **HRR/FHRR `unbind`** is `Empirical` — the approximate inverse is lossy, not self-inverse, and needs cleanup; it is "the residual weak link" (RFC-0003 §4 Net / §4.1, RR-13). HRR/FHRR `bundle` is `Empirical` (Gaussian/asymptotic), upgrading to `Proven` only under the sub-Gaussian phasor side-condition (Thomas) — FLAGGED Q3, not pre-claimed. `cleanup` (and indexed `reconstruct`) is `Empirical`: indexed retrieval is codebook + similarity + threshold returning a *stored atom*, bounded-lossy (Clarkson Thm 16), **not** holographic reconstruction (RFC-0003 §6).
- **Reconstruction is probabilistic-only (FR-C2).** Resonator factorization (`reconstruct_factors`) is capped at **`Empirical`** for a model with an exact bind (MAP-I/BSC) and **`Declared`** otherwise (sparse/HRR/FHRR — their bind is lossy / not exactly self-inverse, RFC-0009 §9 Q6); it is **never `Proven`** — convergence is not guaranteed (Frady et al. 2020; RR-5), the `mycelium-core::recon` schema *rejects* a `Resonator` decode whose basis exceeds `EmpiricalFit` (A6, FR-C2), and this spec must not be read as a basis to relax that ceiling (VR-5). "Converged" ≠ "correct": only a clean `Converged` verdict clearing `τ_lock` + confidence + margin yields factors; `BudgetExhausted`/`Oscillating`/`Stalled`/`OutsideEmpiricalProfile` are explicit error verdicts, never a returned wrong fixed point (RFC-0009 §5/§6).
- **Compositions never upgrade.** `encode_seq`/`encode_set` and compositional `reconstruct` tag at the **meet** of their constituents (weakest-wins; RFC-0001 §4.7) — a sequence built from a `Proven` `bundle` and `Exact` `permute` is no stronger than its `bundle`, and one routed through an `Empirical` `unbind` is `Empirical`.

## 5. §4.1 contract conformance (C1–C6)

- **C1 — never-silent (G2):** every approximate or partial outcome is an explicit, propagating result. A MAP-B nested-bundle input is `Err(NestedBundleUnsupported)` (RR-13 — never a silent accuracy loss). A `cleanup` below its confidence threshold is `Err(BelowCleanupThreshold)`; an ambiguous near-tied slot is `Err(Ambiguous)` — **never a silent low-confidence guess passed off as a match** (FR-C2; RFC-0009 §5.4). A resonator run that does not cleanly converge returns `BudgetExhausted`/`Oscillating`/`Stalled`/`OutsideEmpiricalProfile` (error verdicts), never a populated factor struct on non-convergence (the RFC-0009 §6 / §8.1 P5 correction over the prior art). Out-of-range manifest params are `MalformedReconstruction`, never silently accepted (RFC-0003 §6.1). Mismatched model/dim and an empty bundle are explicit errors, never coercions or fabricated vectors.
- **C2 — honest per-op tag (VR-5):** this is the module's crux. Each `(model, op)` row carries **exactly the tag the RFC-0003 §4 matrix (the M-242-encoded `matrix`) assigns** — no tag is invented, and none is upgraded. `Proven` appears only where the §4 matrix cites a non-asymptotic bound with checked instantiation (the M-131 pattern); HRR/FHRR `unbind` stays `Empirical`; reconstruction stays the probabilistic-only `Empirical`/`Declared` ceiling (FR-C2). The matrix is asserted in tests against `matrix.rs` once code lands (§4.5), so the honesty claim is a checked table, not prose.
- **C3 — no black boxes / EXPLAIN (SC-3/G11):** every selecting/approximating op reifies *why*. A `bundle` carries its `CapacityBound` certificate; an HRR/FHRR `unbind` carries its `CrosstalkBound`; a `cleanup` carries `(confidence, margin)` and, on refusal, a diagnostic record (RFC-0013) naming the violated threshold/margin. A resonator decode is reified end-to-end (RFC-0009 §4): the static manifest (content-addressed codebooks, recipe, decode params) plus a dynamic, `EXPLAIN`-able **run trace** — the per-iteration per-factor top-atom + similarity (the similarity trajectory), the stop reason, the iteration count, and the final confidences. No opaque pass sets a user-visible outcome.
- **C4 — content-addressed, value-semantic (ADR-003 / RFC-0001):** hypervectors are immutable `VSA`-`Repr` values; every op is a pure function of its inputs (a seeded resonator init is a *declared input*, RFC-0009 §6, not ambient state). Codebooks and recipe atoms in the manifest are **content-addressed `ContentHash` refs** (RFC-0003 §6; owned by `std.content`). The carried `Bound`/`Meta` (capacity/crosstalk, measured confidence, decode params) is **metadata — not identity** (ADR-003).
- **C5 — above the small kernel (KC-3):** `std.vsa` **wraps** the landed `mycelium-vsa` (M-130/240/241/242/132) and the `mycelium-core::recon` manifest (M-260); it defines no new `Repr` kind (the four are closed, RFC-0001 §4.1) and no new model, and adds **no** trusted code. The kernel carries only the `VSA` `Repr` kind, the `Hypervector` slot + metadata, the swap machinery, and the `ModelId` registry hook (RFC-0003 §2); the algebra lives in the submodule this module surfaces. No `wild`/FFI (ADR-014) is asserted at the `vsa` layer.
- **C6 — declared, bounded effects (RFC-0014):** every op is **pure** (`effects: none`) — no IO, no clock, no ambient randomness. The resonator's iteration is **bounded** by the manifest `iteration_budget` (`≥ 1`, schema-checked) so a run cannot hang (RFC-0009 §6); its initialisation randomness is a **declared, seeded** input (RFC-0003 §6.1 `seed`), not an undeclared entropy draw — a deterministic run is reproducible.

## 6. Grounding

- **The per-`(model, op)` tags** — RFC-0003 §4 (the normative guarantee-tag matrix, T1.2) **as corrected by the r3 §4.1 erratum** (`permute` `Exact` for all models; HRR/FHRR `bind` `Exact` / `unbind` `Empirical`), the authoritative reading encoded in `mycelium-vsa::matrix` / `tests/matrix.rs` (M-242). The `bundle` Proven/Empirical splits cite Clarkson (Thms 6, 9, 16, 22–23), Thomas (Thms 2/7), Heim / Yi & Achour, and Frady — *cited, not restated*. The `Proven`-needs-checked-instantiation discipline is RFC-0003 §5 + the landed M-131 pattern.
- **The `VsaModel` interface + cleanup** — RFC-0003 §3 (`bind`/`unbind` + self-inverse flag, `bundle` + capacity-bound, `permute`, `similarity`, cleanup integration; per-op tag + basis). Landed: M-130 (`VsaModel` + MAP-I), M-240/M-241/M-242 (MAP-B/BSC, HRR/FHRR, sparse/SBC + `matrix` + RR-13), M-132 (`CleanupMemory` → `(label, confidence, margin)`).
- **The reconstruction manifest** — RFC-0003 §6/§6.1 (indexed retrieval vs compositional reconstruction; the minimal manifest contents; resonator decode params; the probabilistic-only ceiling), the `reconstruction-manifest.schema.json` projection (M-010), and the landed M-260 (`mycelium-core::recon` + `mycelium-vsa::recon`). The `spore` scope note (RFC-0003 §6 r2; ADR-013) draws the manifest-vs-deployable boundary with M-522.
- **The resonator honesty contract** — RFC-0009 (Accepted): probabilistic-only, **never `Proven`** (FR-C2 / G4 / RR-5); the never-silent termination verdicts (`Converged`/`BudgetExhausted`/`Oscillating`/`Stalled`); "converged ≠ correct"; confidence + margin refusals; seeded determinism. The M-350 prototype's *validated envelope* is recorded there (informative), not restated here.
- **The contract + matrix obligation + ring** — RFC-0016 §4.1 (C1–C6), §4.2 (Ring 1 capability surfaces are certificate/EXPLAIN consumers), §4.3 (the `vsa` row: "hypervector models … cleanup memory, reconstruction manifests, resonator decode", grounding RFC-0003/0009; M-130/240–242/260/350), §4.5 (every module ships a checked guarantee matrix).
- **The value model + house requirements** — RFC-0001 (the `VSA` `Repr` §4.1, `Hypervector`, `ModelId`, `CapacityBound`/`CrosstalkBound` §4.3, the lattice + meet-composition §4.7); VR-5 (honest tags, downgrade not upgrade), G2 (never-silent), SC-3/G11 (EXPLAIN), KC-3 (small kernel), ADR-003 (metadata ≠ identity), FR-C2 (reconstruction probabilistic-only).

## 7. Open questions (FLAGGED — resolve before ratification)

- **(Q1) Scope ratification (M-501).** This spec is authored design-first; the `vsa` scope — the exported op set, the encoding-utility surface, and the manifest-construction entry points — is **ratified at M-501 with RFC-0016**, not here. The §3 sketch is illustrative (the template rule); the committed surface is the landed `mycelium-vsa` API, against which §3/§4 row labels must be reconciled at ratification. — *Disposition: FLAGGED; align with the crate API and ratify with RFC-0016. Ties to RFC-0016 §8-Q1 (the v0 module set) / §8-Q2 (naming — `mycelium-vsa → std.vsa`, and the `hdc` alias).*
- **(Q2) `similarity`'s tag — `Exact` score vs `Empirical` decision.** The `similarity` *primitive* is a deterministic score, so it is tagged `Exact` here. But similarity is consumed as a *decision* input (cleanup, the resonator's per-slot argmax), where the **decision** is bounded-lossy. The matrix tags the primitive `Exact` and the *decision* ops (`cleanup`, `reconstruct_factors`) at their lossy tag; whether `similarity` should additionally carry a crosstalk-context bound when used decisionally is open. — *Disposition: FLAGGED; confirm against `mycelium-vsa`'s similarity contract; do not upgrade or conflate the primitive with the decision.*
- **(Q3) Which `bundle`/`bind` cells legitimately reach `Proven`, and the exact theorem instantiation.** The matrix tags HRR/FHRR `bundle` `Empirical` (Gaussian), upgrading to `Proven` *only* under the sub-Gaussian phasor side-condition (Thomas), and sparse/SBC `bind` `Proven` for the *algebraic part*. The **exact constants, norms, and checked side-conditions** are owned by RFC-0003 §4/§5 and the encoded `matrix` (M-242) — **not reproduced here**; any cell the checker cannot discharge **downgrades** (VR-5). — *Disposition: FLAGGED; cite the encoded matrix, never restate or upgrade. Resolve against `matrix.rs` at ratification.*
- **(Q4) Resonator coverage beyond MAP-I/BSC — `Declared`, and the validated regime.** RFC-0009 caps resonator factorization at `Empirical` for exact-bind models (MAP-I first, then BSC) and **`Declared`** for sparse/HRR/FHRR (lossy / non-exact self-inverse, §9 Q6); the validated `{F, ∏kᵢ, d}` regime is *measured by trials* (the `ResonatorProfile`), not asserted. Whether `std.vsa` exports the `Declared` paths in v0 (vs MAP-I/BSC only) is open. — *Disposition: FLAGGED; default to the ratified RFC-0009 coverage (MAP-I/BSC `Empirical`); export the `Declared` paths only behind their explicit `Declared` marker. Ties to RFC-0009 §10.3 (deferred coverage).*
- **(Q5) Ergonomics of the tag/bound/EXPLAIN-carrying return (tension A).** Every op returns value-plus-tag (and bound, where one applies), and a manifest is content-addressed and EXPLAIN-able; how much of that machinery is implicit-but-inspectable vs explicit at the call site is the RFC-0016 §8-Q3 (ergonomics-vs-contract) tension, instantiated for hypervector algebra. — *Disposition: FLAGGED; a per-ring design pass, not a per-op answer here. Ties to RFC-0016 §8-Q3.*
