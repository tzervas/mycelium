# tree-sitter-mycelium

A [tree-sitter](https://tree-sitter.github.io/tree-sitter/) grammar for the Mycelium language
(`.myc`, scope `source.mycelium`) — the structural grammar that powers syntax highlighting,
folding, and other editor-integration features in Neovim, Zed, Helix, Emacs, and any other
tree-sitter-consuming tool.

## Status (transparency — VR-5/G2)

This is a **v2 STRUCTURAL** grammar (RFC-0026 §3.4 follow-up; M-697), not a keyword scaffold: it
covers the full landed surface — declarations (`nodule`/`phylum`/`use`/`type`/`trait`/`impl`/
`object`+`via`/`lower`/`derive`/`fn`+`@tier`), effects (`!{…}`), expressions (`let`/`if`/`match`
with or-patterns/`for`/`swap`/`with`/`wild`/`spore`/`consume`/`colony`+`hypha`+`@forage`/`fuse`/
`reclaim`/`lambda`), the full RFC-0025/M-745 operator-precedence table, repr types (incl. the
RFC-0037 D2-b short aliases `bin`/`tern`/`emb`/`hvec`), guarantee annotations (`T @ Strength`),
and function types (`A => B`, right-associative).

**Guarantee tags:**
- The keyword sets are **mechanical** — lexer-derived from `crates/mycelium-l1/src/token.rs` and
  kept in lockstep by `just drift-check` (never hand-edited here).
- The structural productions are **Empirical** — verified by parsing the full
  `docs/spec/grammar/conformance/accept/*.myc` corpus (25 files) and `lib/std/*.myc` (17 files)
  with **zero ERROR/MISSING nodes**, plus this package's own `test/corpus/` (12 cases). This is
  *not* a proof of equivalence to the normative grammar.
- **`docs/spec/grammar/mycelium.ebnf` stays the accept/reject oracle.** Where this grammar is more
  permissive than the EBNF/L1 parser, it is a **documented, Declared deviation**, never a silent
  one: (1) reserved-not-active keywords (`backbone`/`cyst`/`graft`/`grow`/`matured`/`mesh`/`xloc`)
  parse as an explicit `reserved_keyword` atom, so a highlighter can render them in snippets/docs,
  even though the L1 parser rejects them; (2) repr-type brace arities (e.g. how many args
  `Dense{…}` takes) are checker territory, not enforced structurally here.

## Generated-file policy — do not hand-edit

Two layers of generated artifacts, each with its own regeneration path:

| File(s) | Generated by | Regenerate with |
|---|---|---|
| `grammar.js`, `queries/highlights.scm` | `tools/grammar/generate.py` (lexer-derived, drift-gated against `crates/mycelium-l1/src/token.rs`) | `just grammar-gen` (repo root); verify with `just drift-check` |
| `src/` (`parser.c`, `grammar.json`, `node-types.json`, `tree_sitter/*.h`) | the `tree-sitter` CLI, from `grammar.js` | `tree-sitter generate` (this directory) |

`src/` is **committed** — the tree-sitter convention — so downstream consumers (nvim-treesitter,
Helix, Emacs's `treesit`) can compile the parser without installing the tree-sitter CLI
themselves. Never hand-edit anything under `src/`; regenerate it from `grammar.js` instead.

## Build & test

Requires Node.js and the tree-sitter CLI (no local install needed — `npx` fetches it):

```sh
cd tools/grammar/tree-sitter-mycelium

npx tree-sitter-cli@0.25 generate   # regenerate src/ from grammar.js
npx tree-sitter-cli@0.25 test       # run test/corpus/*.txt — must be green
npx tree-sitter-cli@0.25 parse path/to/file.myc   # inspect a parse tree
```

`test/corpus/` holds tree-sitter's `==== name ==== / code / ---- / S-expression` corpus format —
12 cases derived from the landed conformance corpus + this grammar's own repr/function-type/
attribute forms, each verified against an actual `tree-sitter parse` output (never hand-guessed).

## Consuming this grammar

This package is **not yet npm-publishable** — full node-binding scaffolding (`bindings/node`,
`binding.gyp`, a Rust `Cargo.toml` crate binding) is explicitly **deferred**; only `grammar.js` +
the committed `src/` (the CLI/native-compile path) are provided. For per-editor wiring
instructions (Neovim `nvim-treesitter`, Zed, Helix, Emacs `treesit`), see
`tools/grammar/DISTRIBUTION.md`.

In short: most tree-sitter-consuming editors take a git URL + subdirectory
(`tools/grammar/tree-sitter-mycelium`) pointing at this package and compile `src/parser.c`
themselves — no npm publish required for that path.
