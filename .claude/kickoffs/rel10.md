## Swarm / parallelization pattern

**Serial-on-ADR-023; parallel elsewhere:**

- M-737 (ADR-023 authoring) must be **ratified (Accepted)** before M-738 (the release
  act) — serialize.
- M-735 and M-736 are **disjoint** (`docs/reference/` vs `crates/mycelium-doc/`) — fan
  out in parallel with each other and with M-737 (they do not depend on ADR-023).
- M-738 depends on M-737 (ADR-023 Accepted) **and** on E13-1 + E18-1 being done
  (external deps, not this wave's to satisfy). If those deps are not yet met, M-738
  is a `BLOCKED` stub — do not attempt the release act prematurely (G2 / never-silent).

**Collision surface owned by this kickoff head (never agent-edited by leaves):**

- `docs/adr/ADR-023-*.md` (the M-737 leaf owns this stub; it flags up any shared content)
- `CHANGELOG.md`
- `docs/Doc-Index.md`
- `tools/github/issues.yaml` + `idmap.tsv`
- `docs/api-index/`

**Cross-track deconfliction (with `r10`, the runtime code track):** directory-disjoint — `rel10`
owns the **prose** (`docs/reference/**`, `crates/mycelium-doc/**`, `docs/adr/ADR-023-*.md`); `r10` owns the
runtime **code + its design docs** (RFC-0008 / RFC-0027, `docs/spec/stdlib/runtime.md`). On the shared
release-surface, `rel10` **owns `CHANGELOG.md` and the `docs/adr/ADR-022-*.md` §5 gate table** — fold in
`r10`'s FLAGged entry + its "T3 → gate-met" status, and **integrate the gate last** (after `r10`'s T3
has landed, so the §5 cell + M-738 reflect its final state). `docs/api-index/` is **regenerated by the
integrator on the merged state** (never hand-edited); per-crate API baselines stay disjoint. `rel10`'s
M-736 *reads* `lib/std/**` (lib10's territory) read-only — a content dependency, not a write collision.

**M-738 is a gate-check step, not a build step.** Its task is to verify the ADR-022
Definition of Done is met (every row closed), then record the conscientious release

`v1.0.0` tag, and advance `CHANGELOG.md [Unreleased] -> [1.0.0]`. Never cut the tag
until every gate row is confirmed — never-silent (G2).
