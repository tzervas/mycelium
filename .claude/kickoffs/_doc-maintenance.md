# Kickoff Doc-Maintenance Contract (anti-drift)

> **Every kickoff references this file. Updating the docs its work touches is part of the kickoff's
> Definition of Done — NOT a follow-up.** The point is to prevent doc drift across *sequential*
> kickoffs: each wave leaves the corpus current, so the next wave starts from accurate docs.

When a wave completes, the integrating parent (orchestrator) **reconciles, in the same landing**, every
doc the work touched. A wave is **not done** until these are current (and several are enforced
mechanically by `just check`'s drift gates — treat a drift FAIL as a DoD failure, not a nuisance):

1. **Spec status moves (append-only, house rule #3).** If the wave *implements* a Proposed/Accepted
   spec, move it forward with a landed-basis note: `Accepted → Enacted` once the code is landed +
   stable (never leave a spec at `Accepted` when its implementation has shipped). Step through states
   — never skip to `Enacted`. Preserve prior status text (append-only). Tag honestly (VR-5): if a
   claim isn't checked, it stays `Empirical`/`Declared`.
2. **`CHANGELOG.md`** — one entry for the *net* change (Keep a Changelog 1.1.0, ISO dates), via
   `/changelog`. Editorial waves say so.
3. **`tools/github/issues.yaml`** — flip the wave's `M-xxx`/`Exx` status (`todo`/`in-progress` →
   `done`, or `blocked` with the corrected blocker) with the landed PR/basis. **Never leave a status
   lagging the code** (the reconciliation-audit lesson — issues.yaml status must equal code reality).
   Add/validate `doc_refs:` (`python3 tools/github/doc_refs_check.py`); dedup after any union merge.
4. **Lexicon / memory** — if the surface, keywords, or vocabulary change: `.claude/memory/
   lang-lexicon-syntax.md` (+ the relevant `.claude/memory/*.md`), and `docs/Glossary.md` /
   DN-02/03/06 if a term's status changes (reserved → active, etc.).
5. **Grammar artifacts** — if the grammar/keywords change: regenerate `docs/spec/grammar/mycelium.ebnf`
   (or hand-edit + keep it matching the parser) **and** the editor grammars (`just grammar-gen` →
   `tools/grammar/*`); the `drift` gate enforces this.
6. **Agent index** — if a public API (Rust symbol) changes: regenerate `docs/api-index/`
   (`just docs-index`); the `drift` gate enforces this.
7. **Doc-Index + footers** — if a doc is added/renamed/superseded: update `docs/Doc-Index.md` and the
   per-doc changelog footer.

**The discipline in one line:** *land the code and its docs together; a green `just check` (incl. the
drift gates) + a status/changelog/issues reconciliation is the gate — so the corpus never lags the
kernel, and the next sequential kickoff inherits truth, not drift.*

Cross-track note: where a wave shares a release-surface file it does **not** own (e.g. `CHANGELOG.md` /
`ADR-022` gate table owned by `rel10`, or `docs/api-index/` regenerated by the integrator), it **FLAGs**
its entry up to the owner rather than editing — the owner reconciles once (kickoffs README
§Cross-track deconfliction). Ownership of a shared doc *rises to the nearest shared parent* (CLAUDE.md
fractal file-ownership rule).
