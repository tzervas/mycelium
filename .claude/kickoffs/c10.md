## Scope

Close the residual ADR-021 Gate A work so the Mycelium **Rust core/kernel** can be declared at
1.0.0. This is a **precondition** for full-lang 1.0.0 (DN-25 / ADR-022), not the full-lang gate
itself — the kernel reaching 1.0.0 is a sub-gate that the full-language 1.0.0 inherits.

Four tasks in dependency order:

1. **M-700** — close the deep-review Medium-findings ledger (Gate A2): resolve or explicitly
   defer each remaining Medium with a one-line rationale.
2. **M-701** — WS8 durability surface (Gate A3): `cargo-mutants` green on the trusted base;
   migrate hand-rolled LCG tests to `proptest` with seed rotation; `cargo-fuzz` targets for
   lexer/parser/checker in CI.
3. **M-702** — supply-chain gate (Gate A4): wire `cargo-deny` and `cargo-audit` into
   `just check` (currently skipped) and make them green.
4. **M-703** — cut core/kernel 1.0.0 + move ADR-021 `Accepted → Enacted` (append-only).

## Swarm & parallelization pattern

M-700 and M-702 are **fully independent** (disjoint code/tooling areas) and can run in parallel
as leaf agents. M-701 (WS8 durability) has four sub-tasks (mutants / proptest / fuzz /
coverage-map) that are themselves disjoint-by-crate and can run as a second-level parallel
swarm. M-703 (the release act) runs **after** M-700/M-701/M-702 are all green.

**Collision surface:** `CHANGELOG.md`, `docs/Doc-Index.md`, `tools/github/issues.yaml`,
ADR-021 status footer — orchestrator-owned. Leaf agents do not touch these.

**Parallelism plan:**

```
Wave A (parallel):  M-700 leaf ∥ M-702 leaf
Wave B (parallel):  M-701 sub-tasks (mutants, proptest, fuzz, cov-map) — after M-700
Wave C (serial):    M-703 (release act) — after all Wave A + B green
```
