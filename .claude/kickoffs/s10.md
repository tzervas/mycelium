## Scope

Finish the Mycelium surface language so that real programs and a self-hosted stdlib are
**pleasant to write** — ergonomic, complete, and backed by a ratified grammar. This is the
surface-language arm of the full-language 1.0.0 path (DN-25 Draft / ADR-022 Accepted).

Five tasks (some parallelizable):

1. **M-704** — full HOF: closures, partial application, multi-arg arrows, dynamic fn-flow
   (the RFC-0024 residuals beyond static defunctionalization; the dynamic-HOF leg).
2. **M-705** — operator syntax (RFC-0025): symbolic infix sugar desugaring to word functions,
   precedence/associativity table, frontend-only (no kernel change, KC-3).
3. **M-706** — concrete surface grammar / L3 ratification (RFC-0030): a committed EBNF
   reference grammar + conformance corpus, resolving RFC-0006 Q3/Q8.
4. **M-707** — RFC-0020 L2 surface term language enactment: the richer surface + inference
   (the Accepted-scoped RFC-0020 carve-outs: §4.2 polymorphic instantiation, §4.5 `grow`-derived
   traits, R20-Q1...Q5).
5. **M-708** — surface stabilization: finalize remaining E7-1 generics/traits/effects edge cases
   to a stable, documented surface.

  expressions or the words-only decision is committed normatively.
- RFC-0030 (grammar) is `Proposed` with a complete, updated `mycelium.ebnf` and a
  conformance-corpus update.
- RFC-0020's carved-out sections are enacted or explicitly re-deferred with forward issue refs.
- RFC-0024 §5 residuals are addressed: dynamic fn-flow implemented or `Residual`-documented.
- `just check` is green on the integrated branch (all crates, all gates).
- No surface form in the conformance corpus is silently refused (G2).

## Agent prompt (self-contained brief)

You are running kickoff `s10` — completing the Mycelium surface language for real programs and
a self-hosted stdlib. The repo is `/home/user/mycelium`. Your working branch is
`claude/head/s10`; branch off `dev` (or `main` if `dev` is current) and push before spawning
leaf agents.

**Context:** RFC-0024 (static HOF defunctionalization) is implemented in `mono.rs`; `lib/std/result.myc`
is the first self-hosted nodule. The surface is feature-complete for monomorphic, polymorphic,
trait-bearing, effect-annotated, pure modules. Remaining gaps: dynamic HOF (closures/partial
application, RFC-0024 §5 residuals); symbolic infix sugar (DN-23 recommends hybrid; RFC-0025
is the binding RFC); grammar ratification (RFC-0030, resolves RFC-0006 Q3/Q8); and RFC-0020
carve-out enactment.

**Your five tasks:**

1. **M-704 (dynamic HOF):** Implement the RFC-0024 §5 residuals: closures capturing an
   environment, dynamic fn-flow through `match` arms or data fields, partial application.
   Requires Reynolds defunctionalization (fn-tag sum + `apply` dispatch in L1/L0). KC-3: no
   new L0 node without an RFC — flag immediately if a kernel node is needed.
2. **M-705 (operator syntax):** Implement RFC-0025's decision (DN-23 recommends hybrid): add
   infix operator parsing to `parse.rs`, extend `mycelium.ebnf` with the operator grammar,
   commit the precedence/associativity table in RFC-0025, wire the desugaring (`a + b` to
   `add(a, b)`) as a frontend-only pass with an EXPLAIN record. Conformance fixtures required.
3. **M-706 (grammar ratification):** Update `docs/spec/grammar/mycelium.ebnf` to a complete,
   ratified L3 grammar. Resolve RFC-0006 Q3 and Q8 with committed normative text in RFC-0030.
   Add conformance fixtures for every extended production.
4. **M-707 (RFC-0020 enactment):** Land RFC-0020 carved-out sections: §4.2 polymorphic
   instantiation inference sugar, §4.5 `grow`-derived forms, R20-Q1...Q5. Move RFC-0020 status
   to `Accepted` (append-only) only once all carve-outs are enacted.
5. **M-708 (surface stabilization):** Audit remaining E7-1 edge cases in generics, traits,
   and effects. Document each as `present` (with a test) or `explicitly deferred` (with a
   forward issue ref). Produce a stable-surface declaration document.

**House rules (mandatory):**
- Orchestrator owns: `tools/github/issues.yaml`, `docs/Doc-Index.md`, `CHANGELOG.md`,
  `docs/rfcs/README.md`, `docs/adr/README.md`, `docs/api-index/`. Never touch these.
- `crates/mycelium-l1/src/` is serial-on-L1 — no two leaf agents edit `parse.rs`
  simultaneously. Commit + push the integration before launching the next leaf.
- `just check` must be green before every merge. Report any skip.
- Honesty: RFC-0025 and RFC-0030 are **Draft**; move to `Proposed` after the leaf work is done,
  never to `Accepted` without maintainer ratification. Append-only — never rewrite status lines.
- KC-3: any new node in `mycelium-core` requires a new RFC. Flag up; do not add silently.
